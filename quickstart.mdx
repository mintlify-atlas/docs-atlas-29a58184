title: "Quick start"
description: "Get up and running with Gemma Gem in minutes."
---

Gemma Gem runs Google's Gemma 4 model entirely on-device via WebGPU. Once installed, it adds a gem icon to every page you visit. You can open it to ask questions about the page or ask it to take actions on your behalf.

<Note>
  If you haven't installed the extension yet, follow the [Installation](/installation) guide first.
</Note>

## Get started

<Steps>
  <Step title="Navigate to any page">
    Open Chrome and go to any website. The gem icon appears in the **bottom-right corner** of the page. On first use, a download progress ring is shown on the icon while the model is being fetched and cached.
  </Step>

  <Step title="Wait for the model to download">
    The first time you use Gemma Gem, the selected model is downloaded from Hugging Face. Progress is shown both on the gem icon and inside the chat panel once you open it.

    <Note>
      The default model (Gemma 4 E2B) is approximately 500 MB. Download time depends on your connection speed. The model is cached locally and does not need to be re-downloaded on subsequent uses.
    </Note>
  </Step>

  <Step title="Open the chat">
    Click the gem icon in the bottom-right corner to open the chat overlay. Once the model has loaded, the input field becomes active and you can start typing.
  </Step>

  <Step title="Ask a question or request an action">
    Type a message and press **Enter**. Gemma Gem reads the current page and responds based on its content. You can also ask it to interact with the page directly.

    **Example prompts:**

    ```
    Summarize this page
    ```

    ```
    What are the main topics covered here?
    ```

    ```
    Click the login button
    ```

    ```
    Fill in the search box with "gemma model"
    ```
  </Step>

  <Step title="Stop a response">
    If you want to interrupt a response mid-stream, click the **stop button** that appears in the chat input area while the model is generating. This sends a `chat:stop` message and halts token generation immediately.
  </Step>
</Steps>

## Choose a model

Gemma Gem supports two model sizes. You can switch between them in the settings panel (click the gear icon in the chat header). Your selection persists across sessions.

<Tabs>
  <Tab title="Gemma 4 E2B (default)">
    **Gemma 4 E2B** is the default model and the recommended starting point.

    | Property | Value |
    |----------|-------|
    | Download size | ~500 MB |
    | Context window | 128,000 tokens |
    | Hugging Face model | `onnx-community/gemma-4-E2B-it-ONNX` |

    **Best for:**
    - Getting started quickly
    - Machines with less GPU memory
    - Most everyday browsing tasks: summarizing pages, answering questions, clicking elements

    The E2B model uses q4f16 quantization and supports the full 128K context window.
  </Tab>
  <Tab title="Gemma 4 E4B">
    **Gemma 4 E4B** is the larger, more capable model.

    | Property | Value |
    |----------|-------|
    | Download size | ~1.5 GB |
    | Context window | 128,000 tokens |
    | Hugging Face model | `onnx-community/gemma-4-E4B-it-ONNX` |

    **Best for:**
    - Complex reasoning tasks
    - Machines with more available GPU memory
    - Tasks that require following multi-step instructions precisely

    Switching to E4B triggers a model reload. The extension disposes of the current GPU resources and loads the new model before accepting new messages.

    <Warning>
      Make sure you have at least 1.5 GB of free disk space before switching to E4B.
    </Warning>
  </Tab>
</Tabs>

## Example interaction

Here is a typical session using Gemma Gem on a page:

```
User:  Summarize this page in three bullet points

Gemma: Here's a summary of the page:
       • The article introduces the Gemma 4 model family, covering E2B and E4B variants.
       • It explains how the models run locally via WebGPU without sending data to the cloud.
       • Setup requires Chrome with WebGPU support and approximately 500 MB of disk space.
```

```
User:  Click the "Read more" link

Gemma: [Uses click_element tool]
       Done — I clicked the "Read more" link.
```
